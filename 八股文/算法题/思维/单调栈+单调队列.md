本质一样，都是里面的元素是有序的

#### 本质

对于单调栈，我们每次入栈，要找到之前入栈的第一个满足条件的元素
那入栈的时候，如果对应之前的元素不满足条件，那么**对于后面的元素它也不可能是第一个满足条件**的，就能抛弃掉

对于单调队列，跟单调栈比就是多了个口能漏出来老元素
因为要求多了一个：**超过滑动窗口的元素不能算满足条件**
并且队列内部的元素索引有序，那么就很方便能淘汰所有不满足的元素

#### 模板

```
遍历数组：
	(如果是单调队列：
		如果队头元素超过窗口：
			弹出队头元素
	)
	当栈顶元素不满足当前元素要求：
		出栈
	当前栈顶就是最近的满足条件的元素
	进行相关处理
	入栈当前元素
```

#### 子数组的最小值之和

给定一个数组，定义函数`min(b)`为子数组的最小元素值，求所有子数组的该函数和

我们可以知道，对于一个元素，找到它作为最小值的区间。那么**包含这个数的属于这个区间的子数组的函数为这个元素的值**
那么我们可以遍历每个子数组，不过采用特殊的遍历方式。我们可以知道子数组可以分成多组，对于平时的双层循环，左端点的子数组为一组，然后遍历完每一组就是遍历完了所有子数组。同时，我们把**以最小值为对应位置的元素的来分组，一开始就知道函数值是多少了**

```java
// 找到每个元素左右第一个更小的
// 在这个开区间，包含这个元素的子数组的min(b)确定
int n = arr.length;
// arr[i]左右第一个更小的元素索引
int[][] edge = new int[n][2];
Stack<Integer> stk = new Stack<>();
for(int i = 0;i<n;i++){
	while(!stk.isEmpty() && arr[i]<=arr[stk.peek()]){
		stk.pop();
	}
	edge[i][0] = stk.isEmpty()?-1:stk.peek();
	stk.add(i);
}
stk = new Stack<>();
for(int i = n-1;i>=0;i--){
	while(!stk.isEmpty() && arr[i]<arr[stk.peek()]){
		stk.pop();
	}
	edge[i][1] = stk.isEmpty()?n:stk.peek();
	stk.add(i);
}
long ans = 0;
for(int i = 0;i<n;i++){
	// 遍历所有元素
	// 遍历所有元素作为最小值的子数组，相当于遍历了所有子数组
	int left = edge[i][0]+1, right = edge[i][1]-1;
	long count = (i-left+1)*(right-i+1);
	ans = ((count * arr[i])%MOD+ans)%MOD;
}
return (int)ans;
```

#### 移掉K位数字

给你一个字符串，你可以移除k个数字。找到能产生的最小数

每次肯定是让最高位变小是最好的。那对于每个比较小的数字，它必须要靠前才能让数字变小，所以就删除来让它更靠前。最小的数字一定是前面单调递增的，因为小数前面更大的数字会被移除那么单调栈的性质就是让数字单调递增

实际上就是最小字典序的模板题，最小字典序的真正含义是前面的单调

```java
Stack<Character> stk = new Stack<>();
for(char c:num.toCharArray()){
	while(!stk.isEmpty() && k>0 && c<stk.peek()){
		stk.pop();
		k--;
	}
	stk.push(c);
}
StringBuilder ans = new StringBuilder();
while(k--!=0) stk.pop();
while(!stk.isEmpty()){
	ans.append(stk.pop());
}
ans.reverse();
String s = new String(ans);
int p = 0;
while(p<s.length() && s.charAt(p)=='0') p++;
return p==s.length()?"0":s.substring(p);
```
