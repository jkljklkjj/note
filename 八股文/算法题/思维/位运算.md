对于一个数，它有自己的二进制形式。

#### 本质

对于普通的数，可能就是加减乘除，但是对于二进制数，有特别的运算方式，并且有不一般的结果。可以说位运算就是有这种神奇的魔力

我觉得位运算神奇的本质很简单，因为实际上二进制数就是一个布尔数组，那么多个布尔数组的同位置元素可以算作是同类，同类的元素进行运算后有逻辑的产出

或者可以看成一个集合，比如说6的二进制`1110`，它的第2、3、4位为真，每一位表示一个数从而代表集合`{2,3,4}`。那么位运算就可以很方便来计算集合的运算

#### 加减乘除表

| 符号   | 名称   | 规则             | 通俗理解            |
| ---- | ---- | -------------- | --------------- |
| `&`  | 按位与  | 两个位都是1，结果才是1   | 相当于“两个条件都满足”    |
| `\|` | 按位或  | 只要有一个位是1，结果就是1 | 相当于“至少一个条件满足”   |
| `^`  | 按位异或 | 两个位不相同，结果是1    | 相当于“找不同”        |
| `~`  | 按位取反 | 0变1，1变0        | 相当于“完全反过来”      |
| `<<` | 左移   | 所有位向左移动，右边补0   | 相当于乘以2的幂次       |
| `>>` | 右移   | 所有位向右移动        | 相当于除以2的幂次（向下取整） |

#### 基础

##### 求二进制位数

给定一个数，找到二进制全是1的刚好不小于这个数的数

用到了求一个数有多少位的操作`32 - Integer.numberOfLeadingZeros(n)`（整形为32位4字节）
找有多少位的操作，也可以看成快速找到一个集合的最大数

```java
 int bitLength = 32 - Integer.numberOfLeadingZeros(n);
 return (1<<bitLength) - 1;
```

给定一个数，找到从零到这个数的单比特数（所有位都相同）的数量

可以知道，肯定是0、1、11、111...，那么包含1的单比特数找到这个数的位数-1就可以了。但是如果这个数本身是单比特数那也算数，那么让这个数+1后，单比特数的位数会多一位

```java
int bitCount = 32 - Integer.numberOfLeadingZeros(n+1);
```

##### 求1的个数

给你一个整数数组 `arr` 。请你将数组中的元素按照其二进制表示中数字 **1** 的数目升序排序。
如果存在多个数字二进制中 **1** 的数目相同，则必须将它们按照数值大小升序排列。

```java
return IntStream.of(arr)// 将int[]转成流
	.boxed()// 包装int成Integer来实现比较
	.sorted((a,b)->{// 排序
		int ca = Integer.bitCount(a);// 数1的数量
		int cb = Integer.bitCount(b);
		if(ca!=cb) return ca-cb;
		return a - b;
	})
	.mapToInt(a->a)// 将Integer重新转成int
	.toArray();// 把所有元素收集成int[]
```

#### 集合运算

给你两个正整数 `n` 和 `k`。
你可以选择 `n` 的 **二进制表示** 中任意一个值为 1 的位，并将其改为 0。
返回使得 `n` 等于 `k` 所需要的更改次数。如果无法实现，返回 -1。

这里普通的做法肯定是异或，但是要求k为1的地方n必须为1
那么可以用集合的视角看。那么这里的意思就是k必须是n的子集
对应子集，子集和原集合的交集为子集，或者并集为原集合

```java
return (n & k) != k?-1:Integer.bitCount(n^k);
```

#### 取反码

每个非负整数 `N` 都有其二进制表示。例如， `5` 可以被表示为二进制 `"101"`，`11` 可以用二进制 `"1011"` 表示，依此类推。注意，除 `N = 0` 外，任何二进制表示中都不含前导零。

二进制的反码表示是将每个 `1` 改为 `0` 且每个 `0` 变为 `1`。例如，二进制数 `"101"` 的二进制反码为 `"010"`。

直接模拟即可，首先获取原来数的位数，然后获取对应位数的全1二进制数，然后进行异或即可

```java
if(n==0) return 1;
int w = 32-Integer.numberOfLeadingZeros(n);
return ((1<<w)-1)^n;// 对应位数的全1和原数异或
```

#### 取最后的一

给定一个正整数 `n`，找到并返回 `n` 的二进制表示中两个 **相邻** 1 之间的 **最长距离** 。如果不存在两个相邻的 1，返回 `0` 。

如果只有 `0` 将两个 `1` 分隔开（可能不存在 `0` ），则认为这两个 1 彼此 **相邻** 。两个 `1` 之间的距离是它们的二进制表示中位置的绝对差。例如，`"1001"` 中的两个 `1` 的距离为 3 。

首先我们知道，最后的1000...不影响结果，所以先用`n&(-n)`取最后的1000...，原数与其相除就能去掉后缀0，再除以2来去掉这个1

```java
int ans = 0;
n /= (n & -n) * 2; // 去掉 n 末尾的 100..0
// 和后面的两行其实等效
while (n > 0) {
	int gap = Integer.numberOfTrailingZeros(n) + 1;
	ans = Math.max(ans, gap);
	n >>= gap; // 去掉 n 末尾的 100..0
}
return ans;
```

#### 求是否有且只有一个11

如果其二进制表示中 **恰好** 仅包含 **一对** **相邻的置位** ，则返回 `true` ，否则返回 `false` 。

如10110010，它右移之后变成了101101，那么与之后，只有重复出现的111还剩下1
并且对于11，只剩下了一个1。所以数一下是否只有一个1即可

这里简单来想可以是`Integer.bitCount(m)`，那其实也可以把100-1变成011后与

```java
int m = n&(n>>1);
// return m>0 && (m&(m-1))==0;
return m>0 && Integer.bitCount(m)==1;
```
