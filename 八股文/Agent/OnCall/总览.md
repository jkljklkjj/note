‍「智能 OnCall Agent」项目涉及多种 Agent 技术，主要包括以下几类：Knowledge Index Agent ：负责将用户上传的文档进行向量索引化存储到向量数据库的过程，实现知识的结构化管理和高效检索。

Chat ReAct Agent ：基于 ReAct 模式实现的对话 Agent，用于处理业务咨询场景，支持业务咨询、告警自救、工单预处理等场景的无缝切换。

Plan-Execute-Replan Agent ：即运维 Agent，用于处理告警排查等运维场景，通过计划-执行-再计划的循环机制解决复杂问题。

Multi-Agent 架构 ：项目采用多 Agent 协作模式，通过工具调用实现不同 Agent 间的交互，例如将 AI Ops Agent 的调用封装为工具，供其他 Agent 调用。

技术实现方式 ：图编排工作流 ：通过图编排实现模块化的 Agent 工作流，提升系统的可扩展性和可维护性。ReAct 模式 ：核心流程基于 ReAct 模式实现，无论使用 Java 还是 Go 语言，调用框架 SDK 即可实现 Agent 的核心功能。相关】

技术框架 ：Java 技术栈 ：采用 SpringBoot、SpringAI-Alibaba 等框架。Go 技术栈 ：采用 Goframe、Eino 等框架。辅助技术 ：RAG（检索增强生成） ：增强 Agent 的知识检索能力，提高回答的准确性。MCP（模型控制协议） ：用于多 Agent 协作的协议，确保 Agent 间的有效通信。工具调用机制 ：实现 Agent 与外部系统的交互，例如查询日志、调用 API 等功能。

项目中的多个 Agent 通过统一的后端服务和接口进行管理，实现了知识库 Agent、对话 Agent 和运维 Agent 的协同工作

1. 项目文档rag：把项目文档帮忙上传到rag数据库，然后按需求调用rag查询和用户问题有关的文档并返回
2. 运维agent：通过多方位的知识（项目文档、运行日志、监控、告警群）去推理分析系统现状，并可以采取措施解决系统的问题
3. 对话agent：能够接收用户命令并执行一些功能
