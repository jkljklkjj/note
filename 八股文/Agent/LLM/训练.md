#### Prefix LM 和 Causal LM 区别是什么？

- **Causal LM**：严格自回归，token 只能看左侧历史（下三角 mask）。
- **Prefix LM**：把输入分“前缀区”和“生成区”，前缀区通常可双向可见，生成区仍按自回归方式生成。

区别本质：Prefix LM在条件建模上更灵活，Causal LM在统一生成训练上更直接。

- **Causal LM** (因果语言模型，如 GPT 系列)：使用**严格的因果掩码（下三角掩码）**，即当前位置的 token **只能看到它之前（左侧）的所有 token**，不能看到它之后的任何 token（包括同属输入部分的右侧 token）。它的训练目标是**预测下一个 token**。
    
- **Prefix LM** (前缀语言模型，如 GLM 部分版本)：将输入序列分为 **前缀区 (Prefix)** 和 **生成区 (Generation)**。
    
    - **前缀区**：使用**双向掩码**（类似 BERT），每个 token 可以看到该区域内的所有其他 token（包括左侧和右侧）。这能更充分地理解上下文。
        
    - **生成区**：使用**因果掩码**，按顺序生成，每个 token 只能看到它之前的 token（包括整个前缀区和它左侧的生成区 token）。

也就是说，前者适合续写，也就是生成任务，根据**已有的东西续写后面的**；后者适合执行命令，能很好地**理解命令里面的隐藏关系**，然后执行的过程就用跟前者差不多的操作配合命令要求

### Encoder 编码器与 Decoder 掩码有什么区别？

- Encoder自注意力通常是**双向可见**（除padding位）。
- Decoder自注意力使用**因果mask**，当前位置不能看未来token。

这决定了：Encoder偏理解，Decoder偏生成。

#### 为何现在的大模型大部分是 Decoder-only 结构？

核心原因：

1. 训练目标统一：next-token prediction 可直接吃海量文本。
2. 生成任务天然匹配：对话、写作、代码补全都属于自回归生成。
3. 扩展性强：参数和数据放大时性能提升规律清晰。
4. 工程生态成熟：KV Cache、并行推理框架都围绕 Decoder-only 深度优化。

同时，如果要支持命令理解，可以使用**SFT工程**，就是训练过程的数据使用问答对而不是单纯的文本

#### Transformer 中 multi-head attention 中每个 head 为什么要进行降维？

- 控制计算量与显存
- 每个head在不同子空间学习不同关系
- 保持总维度不变（多头拼接后再投影回去）

#### BERT 非线性的来源在哪里？

主要来自：

- FFN中的激活函数（GELU）
- 注意力中的softmax
- 多层堆叠后残差+归一化形成的复杂非线性表示能力

####  为什么要进行 LN（LayerNorm）？

LayerNorm在特征维度做归一化，使每层输入分布更稳定，缓解训练震荡、加快收敛。相比BatchNorm，LN不依赖batch统计，更适合NLP可变长度与小batch场景
