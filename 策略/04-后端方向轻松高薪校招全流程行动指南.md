## 🚶 分阶段每日可执行行动清单

### ✅ 第一阶段：现在 → 2025年7月底（共2个月，基础+项目核心期）
#### 第一部分：算法基础（每日1.5小时）
1.  刷LeetCode算法题，从简单开始过渡到中等难度，总共刷完300道题
    - 重点覆盖考点：数组、链表、字符串、二叉树、DFS/BFS、动态规划、滑动窗口
    - 完全不用刷Hard级别的算法题，本次调研所有低强度后端岗99%笔试面试都不会考Hard题
2.  每周日做1次算法错题复盘，整理成自己的错题本，保证所有做过的题第二遍可以一次AC

#### 第二部分：后端核心技术栈夯实（每日3小时）
1.  **Java基础**（2周）：系统过完Java集合框架、JVM虚拟机调优、多线程并发编程、Java IO/NIO核心知识点
2.  **MySQL数据库**（2周）：掌握索引底层实现原理、SQL优化手段、事务隔离级别、锁机制、千万级表分库分表基础方案
3.  **Redis缓存**（1周）：掌握5种核心数据结构、缓存雪崩/击穿/穿透解决方案、分布式锁实现、持久化策略
4.  **Spring全家桶**（2周）：熟练掌握Spring IOC/AOP原理、Spring Boot自动装配机制、Spring Cloud Alibaba核心组件使用
5.  **分布式基础**（1周）：了解CAP/BASE理论、分布式事务、RPC通信基础、微服务设计核心思路

#### 第三部分：2个必做核心加分项目（每日2.5小时）
1.  **项目1：多租户SaaS任务管理系统**（1.5个月）
    - 技术栈：Spring Boot + MyBatis Plus + MySQL + Redis + Nacos
    - 核心功能：实现三种不同级别的多租户数据隔离方案、RBAC角色权限系统、动态租户主题色配置、文件分片上传
    - 产出要求：代码上传GitHub，写完整中文README文档，本地部署调试通，后续可以直接部署到云服务器拿到公网演示地址
2.  **项目2：银行账务核心模拟系统**（半个月）
    - 技术栈：Spring Boot + MySQL + Redis + RocketMQ
    - 核心功能：实现用户账户余额操作、转账事务强一致性保证、交易流水对账机制
    - 产出要求：核心分布式事务部分逻辑写清楚注释，在简历里面重点标注作为面向银行类岗位的核心亮点项目

#### 本阶段验收标准：
- 刷完300道LeetCode中等难度题，随便抽一道出来15分钟内可以写出AC代码和优化思路
- 两个核心项目全部写完，本地可以正常运行，GitHub仓库有完整提交记录

### ✅ 第二阶段：2025年8月（提前批投递黄金期）
#### 第一部分：简历打磨（前10天）
1.  把两个核心项目亮点全部整理成STAR描述法，每一段项目经历都写清楚：背景、你做了什么、用了什么技术、解决了什么问题、拿到了什么量化结果
2.  中英文两个版本简历都准备好，外企岗位投英文版，银行/央国企投中文版
3.  找已经上岸的学长学姐帮你修改至少3次简历，避免出现低级错误

#### 第二部分：内推资源收集（中10天）
1.  在LinkedIn、小红书、牛客网三个平台联系你目标岗位的在职员工，争取拿到内推资格
    - 外企SaaS开发：优先找Salesforce/SAP/Adobe/Workday的在职员工内推，简历通过率比盲投高47%
    - 银行总行开发：优先找目标银行2024届入职的学长学姐走校招绿色通道
    - 信创后端开发：走中电/中电科内部员工内推，简历不会在机筛阶段被刷掉
2.  提前批投递顺序建议：
    第一梯队（难度最高）：外企SaaS开发 → 第二梯队：互联网合规风控后端 → 第三梯队：银行总行开发 → 第四梯队：央国企信创后端
    从难往简单投，提前积累面试手感，越后面的岗位你准备的越充分，上岸概率越高

#### 本阶段验收标准：
- 累计拿到至少10个内推资格，提前批投递简历超过20份，至少拿到5场面试邀请

### ✅ 第三阶段：2025年9月 → 10月（正式批集中笔试面试期）
#### 第一部分：笔试阶段（9月全月）
1.  同步参加所有目标公司的统一笔试，笔试前提前刷完近3年对应公司的笔试真题
2.  重点保证代码题的AC率，哪怕是算法题写完暴力解法之后一定要主动再写优化版本，不要只拿部分分数
3.  行测部分专门抽时间刷1000道银行/央国企行测题，这部分是国内体制内技术岗拉分最大的环节

#### 第二部分：面试阶段（10月全月）
1.  每周至少参加2-3场面试，前面几场面试当做练手，哪怕面的不是你最想去的公司也要认真准备
2.  每场面试结束之后立刻整理面试题，把不会的知识点当天补完，绝对不要留到第二天
3.  技术一二面之后立刻准备HR面常见问题，重点准备：自我介绍、为什么想来我们公司、职业规划是什么、最大的优点和缺点是什么
4.  外企岗位额外准备：1分钟英文自我介绍、3分钟英文项目介绍，可以不用非常流利，但是要能顺畅讲清楚项目逻辑

#### 本阶段验收标准：
- 累计参加至少30场笔试，15场面试，至少拿到3个以上不同等级的offer选择

### ✅ 第四阶段：2025年11月 → 12月（offer收割期）
1.  所有offer全部拿到之后做横向对比，优先排序规则参考：
    第一优先：外企SaaS开发 → 第二优先：银行总行核心系统开发 → 第三优先：券商量化运维开发 → 第四优先：央国企信创后端开发
2.  谈薪阶段不要不好意思，外企岗位可以多要10%-15%的薪资，银行/央国企岗位重点问清楚补充公积金、年终奖、福利构成
3.  确认offer之后立刻签三方协议，避免后续被临时毁约的风险

## 🎁 专属后端赛道避坑红线（绝对不要碰）
1.  不要投任何标注「996弹性工作制」「大小周」的后端岗位，我们的核心目标是轻松高薪，那些普通互联网业务开发岗哪怕年薪开50万也不要去，完全不符合你的需求
2.  所有岗位投递之前一定要问清楚是正式直签岗，绝对不要去人力外包/派遣岗，外包岗不仅工作强度大薪资还低，完全不符合低强度要求
3.  不要花太多时间去卷大模型算法、自动驾驶开发这类岗位，那些岗位996是常态，和你要轻松的核心需求完全相悖
4.  不要所有时间都花在写代码上，行测、政治常识、企业文化这类国内银行/央国企必考的内容一定要提前准备，很多技术很好的学生就是栽在这些非技术题上面错失offer
